# 从0到100万用户的扩展

## 1.1 单服务器配置

- 常见所有应用, 缓存(Redis), 数据库DB 都放置在该单台机器上

## 1.2 数据库

- 伴随业务,用户基数的增长. 需要多台服务器性能支撑
  - 一台专用业务流量
  - 一台用作数据库
  - 方便分别进行扩展
- 数据库分类
  - 关系型数据库(RDBMS)
    - MySQL, Oracle, Postgres
  - 非关系型数据库(NoSQL)
    - Neo4j, HBase, MongoDB
    - 一般不支持连接(join)操作

## 1.3 纵向扩展 vs 横向扩展

- 纵向扩展
  - 提升单台服务器性能
    - CPU,RAM etc
    - 适用流量小场景
    - 硬件性能限制
    - 无故障转移能力和冗余
- 横向扩展
  - 向外扩展, 添加多台服务器
  - 大型应用适用

## 1.4 负载均衡器

> 将输入流量均匀分配到负载均衡集里的各个Web服务器中(横向扩展)

- 用户流量达到公共IP的负载均衡器上
  - 用户不会直接访问具体Web服务器
  - **服务器** 间, **服务器** 和 **负载混衡器** 间 通信使用私有IP地址

## 1.5 数据库复制

- 主从数据库模式(常见: 一主多从)
  - 原始数据库(Master 主库)
    - 通常只支持 **写操作**(插入, 更新, 删除)
  - 拷贝数据库(Slave 从库)
    - 保存数据副本且仅支持 **读操作**
- 优势
  - 性能更好
    - 读写分离, 能支撑更多 并行处理查询操作
  - 可靠性高
    - 从数据库可升级主数据库
    - (若唯一)从库宕机, 主库可支持 **读操作**
  - 可用性高

## 1.6 缓存

> 存储响应耗时, 内存热点数据
> 常见缓解 直接查询 数据库 压力

- 使用场景
  - 频繁读, 少写
- 过期策略
  - 一定要有过期, 自动删除时间
- 单点故障(SPOF: signle point of failure)
  - 但缓存服务器会遇到
  - 部署多个数据中心避免单点故障
- 驱逐策略
  - 当缓存已满, 新增环境会删除已有条目
  - 常见策略
    - LRU
    - LFU
    - FIFO

## 1.7 内容分发网络(CDN)

> 地理位置上分散的服务器网络, 用于传输静态内容
> 缓存: 图片, 视频, CSS 和 Js文件等 静态内容

- 注意事项
  - 花销
    - 仅存储常用内容
  - 合理的缓存过期时间
  - CDN回退
    - CDN故障时, 保证服务不中断
  - 作废文件
    - 在文件过期前清理文件
      - 手动调用 API 清理
      - 以最新版本替换老版本, 版本号控制

## 1.8 无状态网络层

> 横向扩展, HTTP 本身无状态
> 会话数据在持久性存储中(DB), Web服务器都通过数据库访问状态数据
