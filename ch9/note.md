# 设计网络爬虫

> 也称为机器人(Bot)或者 Spider, 被搜索引擎广泛地用于发现网络上的新内容或者更新内容.

- 用途
  - 搜索引擎索引
  - 王爷存档
  - 网络挖掘
  - 网络监控

## 9.1 理解问题并确定设计的边界

- 基本算法
  - 给定目标URL, 下载目标URL对应所有网页
  - 从这些网页中提取URL
  - 将新的URL添加到需要下载的URL列表.
  - 重复上述动作
- 相关考量
  - 主要目的
  - 采集数据量规模
  - 采集内容
  - 数据存储需求
  - 重复项处理
  - 封底估算

## 9.2 高层级设计

- 种子URL
  - 本次爬行的起点
  - 划分规则
    - 基于物理地址
    - 基于话题
- URL前线(URL Frontier)
  - 爬行状态
    - 即将下载
    - 已经下载
  - 用来存储即将下载的URL的组件叫URL前线
    - 可以看作一个 FIFO 队列
- HTML 下载器
- DNS 解析器
  - 解析域名为具体IP地址
- 内容解析器
  - 网页下载后, 必须进行解析和校验
- 已见过内容
  - 比较网页的哈希
- 内容存储
  - 大部分内容存储在磁盘中
  - 热门内容存储在内存加速
- URL提取器
  - 从HTML页面中解析和提取链接
- URL过滤器
  - 排除特定内容类型, 文件扩展名, 问题链接和 "黑名单"网站
- 已见过的URL
  - 布隆过滤器和哈希表
- URL存储URL存储
  - 用于保存已访问过的URL

## 9.3 深入设计

- DFS(深度优先搜索) vs BFS(广度优先搜索)
- URL 前线
- HTML 下载器
- 健壮性
- 可扩展性
