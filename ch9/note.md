# 设计网络爬虫

> 也称为机器人(Bot)或者 Spider, 被搜索引擎广泛地用于发现网络上的新内容或者更新内容.

- 用途
  - 搜索引擎索引
  - 王爷存档
  - 网络挖掘
  - 网络监控

## 9.1 理解问题并确定设计的边界

- 基本算法
  - 给定目标URL, 下载目标URL对应所有网页
  - 从这些网页中提取URL
  - 将新的URL添加到需要下载的URL列表.
  - 重复上述动作
- 相关考量
  - 主要目的
  - 采集数据量规模
  - 采集内容
  - 数据存储需求
  - 重复项处理
  - 封底估算

## 9.2 高层级设计

- 种子URL
  - 本次爬行的起点
  - 划分规则
    - 基于物理地址
    - 基于话题
- URL前线(URL Frontier)
  - 爬行状态
    - 即将下载
    - 已经下载
  - 用来存储即将下载的URL的组件叫URL前线
    - 可以看作一个 FIFO 队列
- HTML 下载器
- DNS 解析器
  - 解析域名为具体IP地址
- 内容解析器
  - 网页下载后, 必须进行解析和校验
- 已见过内容
  - 比较网页的哈希
- 内容存储
  - 大部分内容存储在磁盘中
  - 热门内容存储在内存加速
- URL提取器
  - 从HTML页面中解析和提取链接
- URL过滤器
  - 排除特定内容类型, 文件扩展名, 问题链接和 "黑名单"网站
- 已见过的URL
  - 布隆过滤器和哈希表
- URL存储URL存储
  - 用于保存已访问过的URL

## 9.3 深入设计

- DFS(深度优先搜索) vs BFS(广度优先搜索)
  - 整个网络是一个有向图，网页是节点，URL是边
  - DFS的深度可能非常深，更常用是 BFS
  - BFS 使用
    - FIFO队列实现
    - 标准BFS没有考虑URL优先级，需要主动考虑排序规则
- URL 前线
  - 存储待下载URL的数据结构
  - 确保爬虫访问频率
  - 确定URL优先级
  - 有节制的礼貌的访问网站
- HTML 下载器
  - 机器人排除协议 robots.txt
    - 网站和爬虫间的通信标准
    - 君子协定
  - 性能优化方案
    - 分布式爬取
    - 缓存DNS解析器
      - 避免频繁访问DNS服务器发起请求，维护本地映射，定时更新
    - 按照地理位置分布去派发任务
    - 设定超时时间
- 健壮性
  - 一致性哈希
    - 均匀派发流量
  - 保存爬取状态和数据
    - 重启被中断任务
  - 异常处理
  - 数据校验
- 可扩展性
  - 扩展新模块支持需求
- 检测和避免有问题的内容
  - 哈希和校验和
  - 避免无限循环
  - 数据噪声处理

## 9.4 总结

- 扩展讨论
  - 服务器端渲染（动态渲染）
  - 过滤垃圾页面
  - 数据库复制和分片
  - 横向扩展
  - 可用性，一致性和可靠性
  - 数据分析
